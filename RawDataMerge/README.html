Customers Export

Data in the exported files are comma separated and use the double quote (") character as the delimiter.

Note: Not all columns present in the exported files are explained here.

Customers.csv

Email Email of the customer. Configured email notifications will be sent to this email.
Billing Address Line1 Address line 1
Billing Address Line2 Address line 2
Billing Address City The name of the city.
Billing Address State State or Province
Billing Address Country The billing address country of the customer. Must be one of ISO 3166 alpha-2 country code.

Note: If you enter an invalid country code, the system will return an error.

Brexit

If you have enabled EU VAT in 2021 or later, or have manually enable the Brexit configuration, then XI (the code for United Kingdom â€“ Northern Ireland) is available as an option.

Billing Address Zip Zip or postal code. The number of characters is validated according to the rules specified here.
Offline payment method The preferred offline payment method for the customer.
Card Status Current status of the card.
Created At Timestamp indicating when this customer resource is created.
Billing Address State Code The ISO 3166-2 state/province code without the country prefix. Currently supported for USA, Canada and India. For instance, for Arizona (USA), set state_code as AZ (not US-AZ). For Tamil Nadu (India), set as TN (not IN-TN). For British Columbia (Canada), set as BC (not CA-BC).
Customer Portal Status Status of the customer portal.
Net Term Days The number of days within which the customer has to make payment for the invoice.
Taxability Specifies if the customer is liable for tax
Locale Determines which region-specific language Chargebee uses to communicate with the customer. In the absence of the locale attribute, Chargebee will use your site's default language for customer communication.
Billing Address First Name The first name of the billing contact.
Billing Address Last Name The last name of the billing contact.
Billing Address Email The email address.
Billing Address Company The company name.
Billing Address Phone The phone number.
Billing Address Line3 Address line 3
Billing Address Validation Status The address verification status.
Is Location Valid Customer location is validated based on IP address and Card issuing country. If the location is valid, it returns True. If not, it returns False. Applicable only for EU, New Zealand and Australia.
VAT Number Status Represents the VAT validation status. This is applicable if you have configured EU, UK or Australian taxes and the VAT number validation is enabled.
VAT Number Validated Time VAT number recently got validated time
Auto Close Invoice Override for this customer, the site-level setting for auto-closing invoices. Only applicable when auto-closing invoices has been enabled for the site. This attribute is also available at the subscription level which takes precedence.

CustomerComments.csv

Customer Id Identifier of the customer.
Comment Id Unique identifier for the comment.
Notes Actual notes for the comment.
Type Type of comment this is.
Created At The time at which this comment was created
Added By The user who created the comment. If created via API, this contains the name given for the API key used.

CustomerBalances.csv

Customer Id Identifier of the customer.
Promotional Credits Promotional credits balance of this customer.
Excess Payments Total unused payments associated with the customer.
Refundable Credits Refundable credits balance of this customer
Unbilled Charges Total unbilled charges for this customer.
